    通过取消 Windows API 绕过 Cylance 和其他 AV/EDR

    API 挂钩可以与 Web 代理进行比较 - 您的应用程序进行的所有 API 调用（包括它们的参数）（例如CreateFile、、、ReadFile等OpenProcess）都会被 AV/EDR 拦截和检查，然后决定程序的操作/意图是否为是否有恶意。

    EDR 供应商挂钩用户态 API 的方式是通过劫持/修改 Windows DLL 中的函数定义 (API)，例如kernel32/kernelbase和ntdll.

    jmp通过在函数定义的开头插入一条指令来修改函数定义。这些jmp指令将改变程序的执行流程 - 程序将被重定向到 EDR 检查模块，该模块将评估程序是否表现出任何可疑行为，并通过分析传递给 EDR 挂钩/监视的函数的参数来进行评估。这种重定向有时称为detour/trampoline.

    希望下图有助于进一步阐明这一点：

    值得注意的是，并非所有功能都会被 AV/EDR 劫持。通常只有那些已知会在恶意中一遍又一遍地滥用的函数才会上瘾 - thinkCreareRemoteThread和NtQueueApcThread类似的函数。
